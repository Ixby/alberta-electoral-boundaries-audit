DPG-era (Derived Provisional Geometries) artifacts. Superseded by canonical Elections Alberta shapefiles.

These files were generated by tracing electoral boundary polygons from commission PDF thumbnails and calibrating them against 2021 Census DA population data. They were the working substrate for all analysis prior to receipt of official shapefiles (approximately v0.1–v0.8 of the audit pipeline).

| File / Directory | Description |
|---|---|
| `dpg/` | Individual DPG shapefiles and GeoJSON exports, indexed by version |
| `approximate_shape_analysis.md` | Quantification of tracing uncertainty (±500m confidence intervals, Tier A/B/C classification) |
| `build_v7_log.csv` | Step-by-step build log for the v7 DPG release |
| `build_v7_summary.json` | Summary statistics for the v7 DPG topology pass |
| `max_dpi_extract.json` | Commission PDF pixel-extraction parameters at maximum DPI |
| `max_dpi_inspect.json` | Visual inspection results for max-DPI boundary traces |
| `mcmc_100k_and_full_coverage.md` | Notes on the 100k DPG-era MCMC run and full-coverage iteration |

**None of these files should be used in active analysis.** Reference `data/shapefiles/canonical/` instead. The DPG sunset clause in `report_academic.md` §4.1.4 requires all DPG-dependent findings to be recomputed against official shapefiles when disclosed.
